スコアグラフ

1時間に2、3回プールシステムはあなたのサーバから時刻を取得し、現在時刻と比較します。 サーバが応答しないか、時間のずれが100ms以上(監視システムと大まかに比べて)の場合、ポイントが減算されます。 ずれが大きいほど更にポイントは差し引かれます。

サーバのスコアが下がると、グラフの上部にある背景色にどれだけ深刻な状況であるかが反映されます。 表示色は青(ずれは僅か)から黄色、オレンジ色、そして赤(ずれが数秒かサーバに到達不能)と変化します。 グラフは平均値を示すので、色から何が起こったかを必ずしも推測出来るとは限りません。 つまり、グラフは傾向を視覚化するツールに過ぎないということです。 監視システムで検知したことの詳細を正確に知るためには、CSVファイルのリンクをクリックしてダウンロードすることが出来ます。

ずれのグラフ

監視システムは大まかにはSNTP(RFC2030)クライアントのように動作するので、通常のntpdクライアントに比べて、サーバと監視システムの間のランダムなネットワーク遅延の影響をより鋭敏に受けます。 言い換えると、時折発生する大きなずれが生じても心配しないで下さい。 また、ずれを絶対的な性能の測定基準に使用しないで下さい。
